# Git DevOps Workflow Project


## 📌 Project Overview

This project demonstrates the implementation of Git version control best practices in a DevOps workflow. It covers repository management, branching strategies, commits, pull requests, tags, and documentation.

The project was created as part of a DevOps internship task to gain hands-on experience with collaborative development workflows and source code management using Git and GitHub.



## 🎯 Objectives

* Learn Git version control fundamentals
* Understand branching strategies
* Practice creating and managing repositories
* Work with pull requests and merges
* Implement version tagging
* Maintain project documentation using Markdown



## 🛠️ Tools & Technologies

* Git
* GitHub
* Markdown



## 🌿 Branching Strategy

The following branches were used in this project:

* **main** – Production-ready branch
* **dev** – Development branch
* **feature/documentation** – Feature branch for documentation updates

This branching model helps organize development work and follows common DevOps practices.



## 📋 Tasks Performed

### Repository Setup

* Created a GitHub repository
* Cloned the repository locally
* Configured Git workflow

### Branch Management

* Created development branch
* Created feature branch
* Switched between branches
* Managed branch updates

### Documentation

* Created and updated README.md
* Added project information and workflow details
* Maintained documentation using Markdown

### Version Control

* Added and committed changes
* Pushed commits to GitHub
* Managed project history

### Git Best Practices

* Added a `.gitignore` file
* Used meaningful commit messages
* Created version tag `v1.0`



## 🚀 Git Workflow Followed

1. Created repository on GitHub
2. Created a development branch
3. Created a feature branch
4. Implemented documentation changes
5. Committed and pushed changes
6. Used pull request workflow
7. Merged updates into the main workflow
8. Created version tag `v1.0`
